# SuppSwap (Blockbrain AI-first)

Architecture flowchart: see docs/architecture_flow.md.

Current app direction:

- AI-first extraction and parsing through Blockbrain.
- Whole-food replacement options generated by AI using official food-composition references.
- Meal ideas generated by AI only (no local recipe fallback mode in the active flow).
- Dietary filtering and pricing audit remain deterministic support layers.

## Setup

1. Create and activate a virtual environment.
2. Install dependencies.
3. Configure Blockbrain credentials in Streamlit secrets (recommended) or environment variables.
4. Run Streamlit.

### Commands (PowerShell)

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
streamlit run app.py
```

## Keeping the Blockbrain proxy alive

The Continue integration uses `blockbrain_proxy.py` on `http://localhost:4891`.

For an always-on setup on your Windows machine, use the watchdog script:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_blockbrain_proxy.ps1
```

To start it automatically when you log in, register the scheduled task once:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\register_blockbrain_proxy_startup.ps1
```

This keeps the proxy restarted if it exits and brings it back at login.

## Required configuration

Use blockbrain/.streamlit/secrets.toml (recommended) or environment variables.

Primary keys:

- BLOCKBRAIN_API_KEY
- BLOCKBRAIN_BASE_URL (defaults to <https://agentic.theblockbrain.ai>)
- BLOCKBRAIN_AGENT_ID (defaults to researchAgent)

Optional model routing:

- BLOCKBRAIN_ROUTE_MODE: agent_default or model_pinned
- BLOCKBRAIN_MODEL_TEXT
- BLOCKBRAIN_MODEL_VISION

Optional pricing integrations:

- SERPAPI_API_KEY
- DATAFORSEO_LOGIN
- DATAFORSEO_PASSWORD

## Runtime behavior

### Analyze

- Accepts image, barcode, URL, and manual text inputs.
- Uses Blockbrain for vision OCR and text parsing.
- Keeps strict quality gates for extracted input.

### Results

- Maps extracted supplement components to AI-generated whole-food alternatives.
- Dropdown options include nutrient density per 100g and dose-match estimates.
- Dietary profile filtering is applied before final option selection.

### Meals

- Generates selected-food, price-optimized, and macro-optimized meal candidates via AI.
- Coverage logic targets at least supplement-equivalent amounts (exceeding is allowed).
- Shows covered vs uncovered component transparency.

### Pricing

- Uses local price seed plus optional live shopping sources.
- Displays source, confidence, and matching audit details for traceability.
